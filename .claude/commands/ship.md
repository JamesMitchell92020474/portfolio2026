# Ship

Update documentation, commit all changes, and push to GitHub.

## Steps

1. Read the current README.md and review all project files to check if the README needs updating — new pages, features, or to-do items completed since last push.
2. Update README.md if needed to reflect the current state of the project.
3. Stage all changed and untracked files (excluding .claude/settings.local.json).
4. Write a clear, concise commit message summarising what changed.
5. Commit and push to origin main.
